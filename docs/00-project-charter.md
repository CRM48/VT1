# Project Charter

## Project title
Cloud-native architecture evaluation for an agentic AI portfolio-management system

## Project objective
Create the complete design specification and evaluation plan leading into my thesis

## Thesis purpose
Design a cloud-native agentic AI system that autonomously manages a portfolio by continuously processing market data, generating predictions, and making portfolio allocation decisions.
The work specifically focuses on how infrastructure constraints – such as latency, scalability, and reliability – affect the system’s ability to make timely and effective decisions
It evaluates different architectural approaches to optimise operational performance.

## Background
There are many factors which make it much more preferable to deploy to the cloud such as the size of databases required, handling of live data, handling of significant amounts of requests, and so on.

## Primary research question
How do system architecture and infrastructure configuration affect the operational performance of an agentic AI portfolio decision system, and can the resulting system meet its required performance targets?

## Intended deliverables
- Design specification
- Evaluation plan
- Proof-of-concept application
- Preliminary cloud experiments
- Final report
- Presentation

## Current-semester outcome
The design specification and evaluation plan to create an agentic cloud-based portfolio management system which can handle thousands of requests and make informed accurate recommendations

## Future thesis outcome
A portfolio management system in which users can input their current portfolio holdings and obtain recommendations on how they should invest their money to further improve their holdings.
