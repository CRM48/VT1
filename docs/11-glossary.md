# Glossary

| Term                      | Example definition                                                                                                                                             |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Agentic AI                | AI systems that can pursue goals with a degree of autonomy by reasoning, planning, acting, or using tools                                                      |
| Surrogate workload        | A controlled workload used to approximate selected behavior of a more complex future system                                                                    |
| Cloud-native              | An approach to building and operating scalable applications using cloud-oriented patterns such as containers, managed services, and declarative infrastructure |
| Monolith                  | An application deployed as one main unit                                                                                                                       |
| Microservice              | A smaller independently deployable service with a focused responsibility                                                                                       |
| Serverless                | A deployment model where infrastructure provisioning and scaling are largely managed by the cloud provider                                                     |
| Event-driven architecture | A system where components communicate through events                                                                                                           |
| Queue                     | A mechanism for buffering work until a consumer processes it                                                                                                   |
| Idempotency               | The property that processing the same event repeatedly does not produce unintended repeated effects                                                            |
| End-to-end latency        | Time from event submission to completed decision record                                                                                                        |
| Tail latency              | Latency experienced by the slower proportion of events, such as p95 or p99                                                                                     |
| Throughput                | Number of successfully completed events per unit of time                                                                                                       |
| Backlog                   | Work waiting to be processed                                                                                                                                   |
| Cold start                | Additional startup delay when an on-demand execution environment must be initialized                                                                           |
| SUT                       | The system boundary being measured                                                                                                                             |
| Infrastructure as code    | Defining infrastructure through version-controlled configuration files                                                                                         |
| Observability             | The ability to understand system behavior through logs, metrics, and traces                                                                                    |
